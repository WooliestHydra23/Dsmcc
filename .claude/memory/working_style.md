## Core Preferences

### Speed & Execution
**Rule**: "Lightyear speed" - move fast, minimize overthinking, execute decisively

**Why**: Dron values rapid iteration and seeing results. Long planning documents slow things down unnecessarily.

**How to apply**: Once direction is clear, start building immediately. Explain as you go. Ask for approval on major decisions but don't wait for perfection before shipping.

### Planning vs. Doing
**Rule**: Create detailed plans first, then execute without re-planning

**Why**: Dron wants one comprehensive roadmap before diving into work phases. Prevents context-switching and keeps everyone aligned.

**How to apply**: Before Phase 3A/3B/3C, create a single document showing all tasks, timeline, dependencies. Once approved, execute without re-planning each phase.

### Parallel Work
**Rule**: Do things in parallel when possible, not sequentially

**Why**: Efficiency. If something can happen independently, do it simultaneously.

**How to apply**: When building features, batch independent work together. Don't make Dron wait for one thing before starting the next.

### Direct Communication
**Rule**: Be direct and specific, avoid vague suggestions

**Why**: Time is valuable. "Try X approach" is less useful than "Change Y in file Z because..."

**How to apply**: Always include: what to do, where to do it, why it matters, estimated time.

### No Overthinking
**Rule**: Avoid excessive analysis or multiple iterations of the same thing

**Why**: Decision fatigue and reduced momentum.

**How to apply**: When Dron says "continue", assume they've decided and want execution, not more planning.
